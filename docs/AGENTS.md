# Documentation (docs) - Architecture & Agent Guidelines

This directory contains all project documentation.

## 1. Formatting Standards
- Use **GitHub-flavored markdown** for all documentation files.
- Ensure proper use of headings, code blocks, lists, and links.

## 2. No Transient Scratchpads
- Do NOT create temporary notes, scratchpads, or WIP documentation here.
- Only finalized, official, and well-structured documentation should reside in this directory.

## 3. Compatibility Matrix
- The `compatibility-matrix.md` file MUST be kept continuously updated.
- Whenever dependencies, supported Node.js versions, or system requirements change, update the matrix immediately.
